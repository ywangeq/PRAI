# Multimodal Paper System Design

This project uses a `Linear`-inspired design language adapted for a research workbench rather than a marketing site.

## Visual Theme

- Quiet, precise, tool-like
- Dense enough for repeated use
- Low-noise surfaces with one controlled accent
- The graph remains the hero, but the chrome around it should feel disciplined

## Color Roles

- Canvas: `#0b0d12`
- Surface 1: `#111318`
- Surface 2: `#151821`
- Surface 3: `#1b1f2a`
- Hairline: `#252a36`
- Hairline strong: `#343b4d`
- Text primary: `#f5f7fb`
- Text secondary: `#c3cada`
- Text tertiary: `#8a93a8`
- Accent primary: `#5e6ad2`
- Accent hover: `#7683f0`
- Success: `#27a644`
- Warning: `#c47a2c`

## Typography

- Prefer system sans stack for reliability on macOS
- Headings should feel crisp and compact, not editorial
- Body copy should stay at `14px` to `16px`
- Buttons and pills should use medium weight, never oversized type

## Layout

- Main workspace should feel like an application shell, not a landing page
- Use restrained spacing and clear grouping bands
- Cards are allowed for stats, diagnostics, and side panels only
- The right detail pane should read like an inspector panel

## Components

- Toolbar buttons are small, flat, and precise
- Primary action uses the accent fill
- Secondary action uses dark raised surfaces
- Pills should be subtle and information-dense
- Panels use hairline borders and shallow shadows

## Graph Surface

- Graph canvas should feel like a control room surface
- Keep background decoration extremely subtle
- Let node colors carry the semantic variation instead of the page chrome

## Notes

- Keep the graph page aligned with Linear-like product UI
- Keep note pages slightly softer and more reading-friendly

## Do

- Favor clarity over ornament
- Keep interactions crisp and work-focused
- Use one accent consistently

## Do Not

- Do not use large gradients or atmospheric hero visuals
- Do not make controls feel like marketing chips
- Do not let background texture compete with the graph itself
